# BystroMarket: Анализ результатов A/B-тестирования новой платформы

Проект посвящен оценке эффективности новой платформы e-commerce сервиса «BystroMarket». В рамках работы была проанализирована воронка продаж, проверена статистическая значимость изменений конверсии пользователей в добавление товара в корзину, рассчитана величина полученного эффекта с помощью доверительных интервалов, а также разработан интерактивный дашборд для мониторинга метрик.

## Стек технологий
* **Базы данных:** PostgreSQL, SQLAlchemy (выгрузка и интеграция данных)
* **Язык программирования:** Python
* **Библиотеки:** Pandas, NumPy, Matplotlib, Seaborn, SciPy (stats)
* **BI-платформа:** Yandex DataLens

## Этапы исследования

### 1. Выгрузка и предобработка данных
* Настроено подключение к базе данных PostgreSQL с использованием `SQLAlchemy`.
* Написан SQL-запрос для объединения таблиц сырых событий (`raw.events`) и профилей пользователей (`raw.users`) через `JOIN`.
* Проведен парсинг вложенных JSON-структур в поле `metadata` для извлечения стоимости товаров.
* Проверена корректность разбиения пользователей по тестовым группам:
  * **Группа A (Контроль):** 11,676 пользователей[cite: 1]
  * **Группа B (Тест):** 11,707 пользователей[cite: 1]
  * Распределение трафика близко к 50/50[cite: 1].

### 2. Исследовательский анализ данных (EDA) и Воронка продаж
* Рассчитаны базовые метрики активности: среднее и медианное количество событий на одного пользователя[cite: 1].
* Визуализирована динамика активности (количество событий) по дням в разрезе тестовых групп[cite: 1].
* Построена продуктовая воронка по ключевым этапам взаимодействия: `view` (просмотр) -> `click` (клик) -> `add_to_cart` (добавление в корзину) -> `purchase` (покупка)[cite: 1].
* Рассчитаны конверсии как между соседними шагами, так и сквозная конверсия.

### 3. Визуализация и BI (Дашборд в DataLens)
Финальные агрегированные данные были импортированы в Yandex DataLens для создания интерактивной отчетности[cite: 1]. На дашборде реализованы:
* Мониторинг ключевых метрик (активность, объем трафика) в динамике по дням с возможностью фильтрации по тестовым группам.
* Наглядная визуализация шагов продуктовой воронки продаж.
* Сравнительный анализ эффективности групп A и B для оперативного принятия бизнес-решений.

### 4. Статистический анализ (Проверка гипотез)
Для оценки изменения целевой метрики (**конверсии в добавление товара в корзину**) данные были агрегированы на уровне уникальных пользователей[cite: 1].

* **Сформулированные гипотезы:**
  * $H_0$: Конверсия в добавление в корзину между группами A и B не отличается[cite: 1].
  * $H_1$: Конверсия в добавление в корзину в группе B статистически значимо отличается от группы A[cite: 1].
* **Статистический критерий:** Для анализа таблицы сопряженности (бинарный признак: добавил товар в корзину / не добавил) применен критерий **Хи-квадрат Пирсона ($\chi^2$)**[cite: 1].
* **Результат теста:** Получено значение $p\text{-value} < 0.05$, что позволяет уверенно отклонить нулевую гипотезу в пользу альтернативной[cite: 1]. Изменения статистически значимы[cite: 1].

### 5. Расчет величины эффекта
Чтобы оценить не только значимость, но и практическую пользу изменений для метрики `add_to_cart`, были рассчитаны следующие показатели:
* **Абсолютный прирост конверсии (Absolute Lift):** $+4.98\%$ в пользу тестовой группы B[cite: 1].
* **Относительный прирост (Relative Uplift):** $+19.58\%$[cite: 1].
* **95%-й доверительный интервал для разности долей:** составил от $+3.73\%$ до $+6.23\%$[cite: 1]. Так как интервал не включает 0, это подтверждает стабильность и надежность полученного позитивного эффекта[cite: 1].

## Ограничения исследования
* В рамках данного этапа анализ проводился без учета сегментации пользователей по регионам, устройствам и источникам трафика[cite: 1].
* Целевой метрикой теста выступало именно промежуточное действие (добавление в корзину); влияние новой платформы на финальную конверсию в покупку и финансовые метрики (выручка, средний чек) в рамках текущего анализа не оценивалось[cite: 1].
